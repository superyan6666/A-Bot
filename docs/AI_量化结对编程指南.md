# 🤖 AI 量化结对编程与 MCP 使用指南

在本项目中，我们不仅拥有云端运行的自动化量化引擎 `A-Bot`，还在本地配置了直连金融数据的 **Tushare MCP Server**。这使得 AI 助手（如 Cursor / Windsurf / Claude）能够突破大模型的静态知识库限制，实时获取 A 股真实盘口与历史数据，充当你的**全天候专属量化分析师**。

本指南收集了最高效的“AI 使唤技巧”，教你如何榨干这套架构的价值。

---

## 🎯 核心理念：所见即所得的量化开发

以往的量化开发流程是：你想一个因子 -> 写代码 -> 跑回测 -> 发现报错 -> 修改代码。
现在的量化开发流程是：**你提出因子 -> AI 通过 MCP 直接拉取全市场数据现场算给你看 -> 确认有效 -> AI 自动将策略固化进 `main.py` -> 自动推送到 GitHub。**

---

## 💡 高阶使用技巧与 Prompt 模板

### 玩法 1：秒级个股“防雷与体检” (个股深度诊断)
在你手动买入或复盘某只个股之前，不要自己去看软件，直接让 AI 带着量化思维帮你盘它。

> 🗣️ **提示词模板：**
> "调用 MCP 工具获取 [股票名称/代码] 最近 [N] 天的日线走势和今天的基本面。用项目里的 VCP 模型和量能特征帮我诊断一下，看看有没有主力建仓的痕迹？市盈率和量比指标健康吗？"
> 
> 🌰 **示例：**
> *"帮我查一下 万科A (000002.SZ) 最近 10 天的走势，结合今天基本面看看有没有雷。"*

### 玩法 2：纯粹的数据探索与因子挖掘 (策略先导验证)
在往代码里写死某个过滤条件之前，先让 AI 拉真实数据算一算，防止条件过于严苛导致选不出股票。

> 🗣️ **提示词模板：**
> "我想研究一下 [某某策略/因子] 的有效性。请你调用 MCP 拉取全市场今天的基本面数据，算一下如果要求 [条件1] 且 [条件2]，全市场大概会筛选出多少只股票？这些股票集中在什么板块？"
> 
> 🌰 **示例：**
> *"我想在预筛里面加一个『低波红利』过滤。你先拉一下全市场基本面，算算如果 PE < 15 且 PB < 1，全市场还有几只符合？"*

### 玩法 3：盘后极致数据复盘 (寻找遗漏的牛股)
利用 AI 强大的计算能力和模式识别能力，直接从原始数据里大海捞针。

> 🗣️ **提示词模板：**
> "帮我拉取 [股票名称/代码] 过去 [N] 个交易日的数据。帮我找找看它这段时间有没有出现过 [特征A] 且 [特征B] 的极致形态？"
> 
> 🌰 **示例：**
> *"帮我看看 宁德时代(300750.SZ) 过去 30 天，有没有出现过『量能极度萎缩』但『股价完全不跌』的冰点时刻？分别是在哪几天？"*

### 玩法 4：架构师视角的代码生成
让 AI 根据最新的真实数据格式，直接安全地重构代码。

> 🗣️ **提示词模板：**
> "我需要在 `main.py` 中增加对 [某个指标] 的处理。你先用 MCP 获取一下测试数据，确认一下字段名和数据类型，然后再帮我改写 `vectorized_prescreen` 函数，并使用 Unified Diff 格式输出代码。"

### 玩法 5：重构顺序与回滚策略 (架构师级重构操盘)
对于涉及核心链路的深度改造（如打分引擎、数据代理、网络拦截器），严禁让 AI 一次性大包大揽。请命令 AI 严格遵循以下重构顺序与提交流程，确保每一步具有**原子性**与**可回滚能力**：

1. **第一步：先建防线 (配置层提取)**。优先将散落的魔法值和环境变量提取为 `AppConfig` 等单例，提供安全的依赖注入与默认回退，在启动时通过摘要打印（隐藏 TOKEN）验证。
2. **第二步：再理外围 (拦截器与网络)**。独立修改如 Monkey Patch 等外围组件，缩小拦截范围（如基于域名白名单），使用 mock 或 debug log 验证原生请求已被正确隔离与放行。
3. **第三步：最后动刀 (核心逻辑解耦)**。在防线与外围稳固后，再对核心引擎（如因子列表）开刀，将其外置为独立文件（如 `factors_config.py`）。要求通过离线冒烟测试（因子一致性校验）验证替换后的 reasons 完全一致。
4. **提交纪律 (PR 隔离)**：明确要求 AI 每完成上述一个独立阶段，必须单独提交一个带有清晰 message 的 Commit/PR，并且在提交前必须说明测试结果（例如："【已验证】配置加载正常"，"【已验证】白名单放行正常"），绝不揉成一团提交。

## ⚠️ 注意事项

1. **Token 限制**：Tushare MCP 中的 `get_daily_basic` (全市场基本面) 接口需要 Tushare 账户有 **2000 积分** 的权限。如果积分不足，AI 会在调用时收到明确的失败提示。
2. **频率控制**：虽然 MCP 很方便，但不要让 AI 一次性拉取几百只股票的历史日线（这会触发 Tushare 每分钟调用次数限制）。推荐一次诊断 1-3 只个股，或直接拉取**单日全市场截面数据**。
3. **隔离机制**：MCP 仅用于本地开发、投研与聊天交互。云端的 GitHub Actions 将完全依赖于独立的脚本逻辑运行，两者互不干扰。
